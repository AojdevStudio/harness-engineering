# Specs Index

GitHub Issues are the primary system of record for executable work. Issue templates under `.github/ISSUE_TEMPLATE/` provide the structured fields agents need: description, acceptance criteria, verification steps, dependencies, reproduction, and agent-slop correction.

`progress.json` is the lightweight in-repo progress surface for Symphony-style waves. Keep it focused on milestone state and issue references; do not duplicate full issue bodies.
